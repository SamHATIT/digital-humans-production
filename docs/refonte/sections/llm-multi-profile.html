Fallback chain — par profile

Chaque profile définit sa propre chaîne de secours en cas d'échec. Le profile on-premise ne tombe jamais sur du cloud (garantie d'airgap).

ProfilePrimary → fallback 1 → fallback 2
cloudAnthropic → OpenAI → (échec)
on-premiseOllama local → (échec, PAS de cloud)
freemiumOpenAI → Anthropic (cap coût mensuel)

Continuation automatique

Porté depuis V1 (CRIT-02). Quand l'API LLM retourne stop_reason="max_tokens", le router relance automatiquement avec <réponse tronquée> continue en prompt, jusqu'à N itérations (configurable). Transparente pour l'agent appelant — essentielle pour les sorties Marcus (architecture 15k+ tokens) et Emma (SDS write 20k+ tokens).
