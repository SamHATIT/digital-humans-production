La table actuelle est seedée à 2 000 cr/mois (calibrée pour l'ancien modèle "tout Sonnet sans Marcus Opus") — à bumper dans la migration 010 quand on activera le tier gating dans le router LLM.

Hook dans le LLM router

Avant chaque appel LLM, le router appelle credit_service.preflight_check(user_id, model, max_tokens) qui :

  1. Vérifie que le tier de l'utilisateur autorise le modèle (cf. model_pricing.allowed_tiers)
  2. Vérifie que le solde de crédits couvre l'estimation maximale (input + output)
  3. Lève ModelNotAllowedError ou InsufficientCreditsError avant tout appel API

Après l'appel, credit_service.charge() débite le solde réel (basé sur les tokens effectivement consommés) et inscrit une ligne dans credit_transactions avec lien vers execution_id et project_id.
